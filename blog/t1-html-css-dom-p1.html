What are HTML, CSS and the DOM -
non technical description

  • The HTML :

    • The HTML is the structure of the webpage. We can think about it as the skeleton. The browser will interpret the HTML without CSS file linked to it with basic styling (font, different margin and size from title to text for exemple).

  • The CSS :

    • The CSS is its looks and shape. We can think about it as the size of the muscles, the color of the hair, skin...

  • The DOM :

    • The DOM is the tree interpretation of the HTML code by the browser. It looks like a family tree with bigger elements followed by their children (reflecting the identation of the code).

What is meant by boxifying design?

Boxifying the design of a webpage helps to understand its HTML structure. Every element is a box, that can contain other boxes. By boxifying it in hour head (or on paper, or on a graphic visualisation program) we can see its main branches (in the DOM) and following those to possibly try to recreate it ourselves.

What is the box model?

The box model is thinking of your tag element as a box with multiple layers. The first outside layer is the margin, then you get the border, then the padding and then your actual content (text for exemple). It may seem like a lot of unuseful layers that no-one would use, but it is actually very important visualy. Without CSS properties of margin, border, padding, the brower would define a style for them anyway because they are necessary for a confortable viewing.
