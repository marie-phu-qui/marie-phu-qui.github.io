What is grid based design?

Grid based design sounds like the box model, but on the CSS vision of things. It consists on breaking websites into grids of 12 columns. A few grid systems are more popular than others. A smaller on the left and larger on the right is quite popular. Two, Three, Four columns is quite common as well. Having a total of 12 columns allows you to fit those popular options : that is why most of the grids systems use 12 columns. With thoses classes we can better control the overall layout of the webpage.

What is all the hype about responsive webpages?

Today we have many devices to access the internet - it means a lot a variation on screen size (resolution in pixels is what we care about). The property max-width : ...px makes sure that the page will probably look great on every device, but it would need to be centered. Responsive design works with percentages or with the unit we discovered on 'em'.

What is semantic structure?

The semantic structure refers to tags that will be recognised by both computers and humans. We can think of headers h1...h4, image img or the form form. Concerning the header tag, which often refers to titles, it is important to think that the size of text is not the one that matter, because sizing is only styling, what matters is the importance (or structure) of it in the page to be read by the browser on the DOM.

Any other points you found interesting and want to reflect on :

  • .row {display:flex; flex-wrap :wrap;}

    • As a property to be sure to align your columns and not stack them on top of each other.

  • element{overflow:auto;}

    • As a property is a pretty nive feature because it allows you to contain a lot of text in a small area where you can scroll up/down inside the element but not scrolling the whole page. It is visualy satisfying to be able to do that.

  • @media only screen and (max-width; 300px){p{property;...;}

    • This is a beautiful option ! From what my grasp was of this, the @media forms a media query which allows you to change the CSS properties depending on the devices resolution we are reding it from.

  • normalize.css

    • With many many browsers your website may not look the same and be interpreted the same through its different uses. Adding the link : 

    <link rel="stylesheet" src="//normalize-css.googlecode.com/svn/trunk/noralize.css">
    to the head of your document you make sure that every browser will interpret your page the same way. This gives me comfort.
